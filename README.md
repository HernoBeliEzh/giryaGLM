# 🏋️ ГиряGLM

**Лёгкая программа для смены аккаунтов в ZCode desktop**<br/>
с показом лимитов GLM-5.2 и GLM-5-Turbo

</div>

---

ГиряGLM — небольшое настольное приложение (Electron), которое позволяет быстро
переключаться между несколькими аккаунтами ZCode и сразу видеть остаток
токенов по моделям **GLM-5.2** и **GLM-5-Turbo**. Удобно, когда у вас несколько
бесплатных Start-планов и хочется без танцев сменить «севший» аккаунт на свежий.

## ✨ Возможности

- **Импорт текущего аккаунта** в один клик — читает JWT прямо из активной OAuth-сессии ZCode.
- **Ручной ввод JWT** — для добавления любого токена.
- **Смена аккаунта в один клик** — корректно работает при входе через **chat.z.ai (OAuth)**: закрывает ZCode, подменяет сессию, запускает заново.
- **Лимиты в реальном времени** — `billing/balance` Z.ai, с прогресс-баром и временем сброса.
- **Управление аккаунтами** — переименование и удаление.
- **Активный аккаунт** подсвечивается автоматически по совпадению с текущей сессией ZCode.
- **Безопасность** — резервные копии `credentials.json` и `config.json` перед каждой сменой.

## 🎨 Интерфейс (v1.1+)

Компактная **сетка аккаунтов по 5 в ряд** — все аккаунты и их лимиты видны
одновременно, без лишних кликов. На узких окнах сетка адаптивно сворачивается
до 4 → 3 → 2 колонок.

- **Минималистичные блоки** вместо раздутых строк: имя аккаунта, две иконочные
  кнопки (переименовать / удалить), кнопка «Запустить» и прогресс-бары лимитов
  по каждой модели.
- **Лимиты показываются сразу** при открытии — запрос к `billing/balance` уходит
  параллельно для всех добавленных аккаунтов.
- **Тёмная фиолетово-navy палитра** с градиентным акцентом: фон `#0a0a12`,
  карточки `#13131e`, акцент `#7b68ee`. Прогресс-бары меняют цвет по нагрузке:
  фиолетовый (норма) → оранжевый (`≥70%`) → красный (`≥90%`).
- **Активный аккаунт** выделен градиентной подсветкой и боковой полосой.

## 🔧 v1.2.1 — починка HTTP 429 при загрузке лимитов

В версиях до 1.2.1 при нескольких добавленных аккаунтах запрос лимитов
параллельно летел сразу для всех карточек. Сервер Z.ai режет баланс-эндпоинт
**по IP** (не по аккаунту) — поэтому 5+ одновременных запросов почти всегда
получали **HTTP 429 (Too Many Requests)**, и карточки показывали ошибку вместо
лимитов.

В v1.2.1 ГиряGLM:

- **Последовательная очередь** запросов лимитов (`CONCURRENCY=1`) — следующий
  аккаунт опрашивается только после ответа предыдущего. Параллельности больше
  нет, 429 по превышению параллельности не возникает физически.
- **Retry с экспоненциальным backoff** при 429/5xx (1.5с → 3с → 6с, потолок 15с)
  с учётом серверного заголовка `Retry-After`, до 4 попыток.
- **Кеш лимитов на 60 секунд** — повторный опрос того же аккаунта в течение
  минуты не делает новый запрос к Z.ai.
- **401/403 не ретраятся** (невалидный токен — повтор бессмысленен).
- Кнопка **«Обновить»** теперь принудительно обходит кеш и перезапрашивает
  лимиты; в карточке видно источник данных («обновлено» / «из кеша Nс назад»).

## 🔧 v1.2.0 — починка переключения аккаунтов

В версиях до 1.2 смена аккаунта **не работала** при входе в ZCode через
chat.z.ai (OAuth). Причина: ZCode хранит активную сессию не в `config.json`,
а в **зашифрованном** `credentials.json`, и при запуске перезаписывает
`config.json` из него — поэтому подмена `apiKey` в `config.json` ничего не давала.

В v1.2 ГиряGLM:

- Реверс-инжинирит схему шифрования ZCode `enc:v1` (`aes-256-gcm`, ключ
  выводится из `platform` + домашнего каталога + имени пользователя).
- При смене аккаунта подменяет `zcodejwttoken` в `credentials.json` (главное!)
  и дублирует ключ в `config.json` как подстраховку.
- Определяет активный аккаунт из `credentials.json` (источник правды), с
  фолбэком на `config.json`.

Бэкапы `credentials.json.bak` и `config.json.bak` создаются автоматически.

## 📥 Установка

### Из релизов (проще всего)
1. Откройте раздел **[Releases](../../releases)** этого репозитория.
2. Скачайте `ГиряGLM Setup 1.2.1.exe` (инсталлятор) или `ГиряGLM-Portable-1.2.1.exe` (без установки).
3. Запустите — готово.

### Требования
- **Windows 10/11 (x64)**
- Установленный **[ZCode desktop](https://z.ai)** — переключатель работает с его `credentials.json` / `config.json`.

## 🛠️ Как это работает

При входе через chat.z.ai ZCode хранит активную сессию (OAuth) в зашифрованном
файле, а `config.json` дублирует ключ для провайдера:

```
%USERPROFILE%\.zcode\v2\
  credentials.json    →  enc:v1:iv.tag.ciphertext  (zcodejwttoken, user_info, …)
  config.json         →  provider["builtin:zai-start-plan"].options.apiKey
```

Смена аккаунта = **закрыть ZCode → подменить сессию в `credentials.json` +
дублировать ключ в `config.json` → запустить ZCode.exe**.

Шифрование `enc:v1` — `aes-256-gcm`, ключ `sha256` от строки
`zcode-credential-fallback:{platform}:{homedir}:{username}`
(переменная окружения `ZCODE_CREDENTIAL_SECRET`, если задана). Это позволяет
ГиряGLM шифровать значения тем же ключом, что использует ZCode.

Помимо смены, программа запрашивает лимиты:

```
GET https://zcode.z.ai/api/v1/zcode-plan/billing/balance?app_version=3.1.2
Authorization: Bearer <jwt>
```

Ответ содержит `total_units`, `used_units`, `remaining_units` для каждой модели.

## 🔐 Безопасность и приватность

- Все токены хранятся **локально** в `accounts.json` внутри каталога `%APPDATA%\girya-glm`.
- Программа ничего не отправляет вовне, кроме одного запроса к официальному API Z.ai (`zcode.z.ai`) для получения лимитов — и только с вашим собственным JWT.
- Перед каждой перезаписью создаются резервные копии `credentials.json.bak` и `config.json.bak`.
- Программа **не изменяет** и **не удаляет** ваши данные ZCode, кроме подмены полей `zcodejwttoken` (credentials) и `apiKey` (config) при явной смене аккаунта.
- Шифрование `credentials.json` выполняется тем же ключом, что использует ZCode на этой машине — без выноса секретов наружу.

## 🧑‍💻 Разработка

```bash
# Установка зависимостей
npm install

# Запуск в режиме разработки
npm start

# Сборка инсталлятора (NSIS) → release/
npm run dist

# Сборка portable-версии (один .exe без установки)
npm run dist:portable
```

### Генерация иконки
Иконка (`build/icon.ico`) генерируется детерминированно скриптом без внешних
зависимостей:

```bash
node build/icon-gen.js
```

### Структура проекта
```
main.js              # главный процесс Electron + IPC
preload.js           # безопасный мост contextBridge
lib/
  credentials.js     # шифрование enc:v1 и подмена credentials.json ZCode
  config.js          # чтение/запись config.json ZCode (с .bak)
  limits.js          # запрос billing/balance, декодирование JWT
  zcode.js           # taskkill + запуск ZCode.exe
  store.js           # хранилище accounts.json
renderer/
  index.html  styles.css  app.js   # минималистичный flat-интерфейс
build/
  icon.ico  icon-gen.js             # иконка и её генератор
```
